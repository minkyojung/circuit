## 📖 왜 필요한가요?

### 현재 상황 (Before)

웹 개발할 때 버튼 색을 바꾸려면:

```
1. Chrome에서 웹사이트 열기
2. F12 눌러서 개발자 도구 열기
3. 버튼 검사하기
4. CSS 코드 복사
5. VS Code로 돌아가기
6. AI 채팅 열기
7. "이 버튼 색 바꿔줘" + CSS 붙여넣기
8. AI 응답 기다리기
9. 코드 복사
10. 파일 찾아서 붙여넣기
11. 저장
12. Chrome으로 돌아가서 확인

총 12단계, 약 5분 소요, 창 3개 사용
```

### 새로운 방식 (After)

```
1. Octave 브라우저 탭 열기
2. 🔍 버튼 클릭 (Inspect Mode)
3. 버튼 클릭
4. [Add to Chat] 클릭
5. "초록색으로 바꿔줘" 입력
6. AI가 자동으로 코드 수정
7. 브라우저 자동 새로고침 → 완료!

총 7단계, 약 30초 소요, 창 1개 사용
```

**90% 시간 단축!**

## 🌟 핵심 기능 3가지

### 1. 브라우저 탭 (중앙 영역)

**뭔가요?**
- 파일 탭 옆에 "브라우저" 탭 추가
- localhost:3000 같은 개발 서버를 Octave 안에서 바로 볼 수 있음

**왜 좋은가요?**
- Chrome 켤 필요 없음
- Hot Reload 자동으로 작동 (코드 수정하면 자동 새로고침)
- 코드 옆에서 바로 결과 확인

---

### 2. Element Inspection (요소 선택)

**뭔가요?**
- 🔍 버튼 누르고 웹사이트의 버튼, 텍스트, 이미지 등을 클릭
- 클릭한 요소의 모든 정보(HTML, CSS, 크기 등)를 자동으로 추출

**어떻게 작동하나요?**

```
Step 1: 🔍 클릭
   → Inspect Mode 켜짐

Step 2: 버튼 위에 마우스 올림
   → 파란 테두리가 버튼을 감쌈
   → 툴팁에 크기, 여백 등 표시

Step 3: 버튼 클릭
   → Inspector Panel에 버튼 정보 표시
   - 스크린샷
   - HTML 코드
   - CSS 스타일
   - 크기와 여백

Step 4: [Add to Chat] 클릭
   → 채팅창에 버튼 정보 추가됨
```

**왜 좋은가요?**
- CSS 복사/붙여넣기 필요 없음
- AI가 정확한 정보를 받아서 실수 안 함
- 스크린샷도 자동으로 포함됨

---

### 3. 🐛 Console Logs (새 기능!)

**뭔가요?**
- 브라우저 콘솔에 찍히는 로그(에러, 경고 등)를 자동으로 수집
- 채팅 입력창의 🐛 버튼 클릭하면 모든 로그를 AI에게 전달

**어떻게 작동하나요?**

```
Step 1: 웹사이트에서 버튼 클릭
   → 에러 발생! (빨간 글씨)

Step 2: Inspector Panel의 Console 탭 확인
   → "Cannot read property 'name' of undefined"
   → "App.tsx:42" (파일과 라인 번호)

Step 3: 채팅 입력창의 🐛 버튼 클릭
   → 모든 콘솔 로그가 Context로 추가됨

Step 4: "이 에러 고쳐줘" 입력
   → AI가 로그 분석
   → 자동으로 코드 수정
   → 에러 해결!
```

**왜 좋은가요?**
- 에러 메시지 복사/붙여넣기 필요 없음
- 한 번에 모든 로그(15개, 20개)를 AI에게 전달 가능
- AI가 로그를 보고 정확한 수정 제안

## 🎬 실제 사용 시나리오

### 시나리오 1: 버튼 색상 바꾸기

```
1. Octave에서 프로젝트 열기
   → 브라우저 탭에 localhost:3000 자동으로 열림

2. 브라우저 탭 클릭
   → 내가 만든 웹사이트 보임

3. 🔍 버튼 클릭
   → Inspect Mode 켜짐

4. 바꾸고 싶은 버튼 클릭
   → Inspector Panel에 버튼 정보 표시
   - 현재 색: 파란색 (#3b82f6)
   - 크기: 120px × 48px
   - 여백: 12px 24px

5. [Add to Chat] 클릭
   → 채팅에 버튼 이미지 나타남

6. "이 버튼 초록색으로 바꿔줘" 입력
   → AI: "Button.tsx 수정했습니다!"
   → 브라우저 자동 새로고침
   → 버튼이 초록색으로 바뀜!

완료! (30초 소요)
```

---

### 시나리오 2: 에러 디버깅

```
1. 브라우저에서 "Submit" 버튼 클릭
   → 에러 발생! (아무것도 안 됨)

2. Inspector Panel - Console 탭 확인
   → ❌ "Cannot read property 'name' of undefined"
   → ⚠ "React Hook useEffect missing dependency"
   → ⓘ "User clicked submit button"

3. 채팅 입력창의 🐛 버튼 클릭
   → "Console Logs (15 messages)" Context Pill 추가됨

4. "이 에러들 고쳐줘" 입력
   → AI: "App.tsx 42번 라인 확인했습니다."
   → AI: "user 객체가 null일 수 있어서 에러 발생"
   → AI: "방어 코드 추가했습니다"
   → 브라우저 자동 새로고침
   → 에러 해결!

완료! (1분 소요)
```

---

### 시나리오 3: 레이아웃 조정

```
1. 웹사이트 보니까 "Cancel"과 "Submit" 버튼이 너무 붙어있음

2. Inspect Mode로 "Cancel" 버튼 클릭
   → [Add to Chat]

3. "Submit" 버튼도 클릭
   → [Add to Chat]

4. 채팅에 두 개의 버튼 이미지 표시됨

5. "이 두 버튼 사이에 간격 좀 넣어줘" 입력
   → AI: "첫 번째 버튼에 margin-right: 16px 추가했습니다"
   → 브라우저 자동 새로고침
   → 버튼 사이에 간격 생김!

완료!
```

## 🖥️ 어떤 프로젝트에서 사용할 수 있나요?

### ✅ 완벽하게 작동하는 것들

| 프레임워크 | Dev Server 주소 | 작동 여부 |
|-----------|-----------------|-----------|
| **React (Vite)** | localhost:5173 | ✅ 완벽 |
| **Next.js** | localhost:3000 | ✅ 완벽 |
| **Vue** | localhost:8080 | ✅ 완벽 |
| **Angular** | localhost:4200 | ✅ 완벽 |
| **Svelte** | localhost:5000 | ✅ 완벽 |

**이유**: 이것들은 모두 **웹 기술**이라서 브라우저에서 그대로 볼 수 있어요!

---

### ⚠️ 부분적으로 작동하는 것들

#### **Electron (데스크톱 앱)**
- **작동**: 웹 UI 부분만 (HTML, CSS, React 등)
- **안 됨**: 파일 시스템, 네이티브 창 관리
- **권장**: UI는 Octave로, Native 기능은 실제 앱으로 테스트

#### **React Native + Expo**
- **작동**: `expo start --web` → 웹 버전 미리보기 가능
- **안 됨**: 실제 iOS/Android 화면
- **권장**: 웹 버전으로 레이아웃 확인, 실제 테스트는 시뮬레이터

---

### ❌ 안 되는 것들

| 플랫폼 | 이유 |
|--------|------|
| **iOS (Swift)** | iPhone 시뮬레이터 필요 |
| **Android (Kotlin)** | Android 에뮬레이터 필요 |

**왜 안 되나요?**
- Octave 브라우저는 **웹만** 볼 수 있어요
- iOS/Android는 **네이티브 앱**이라서 시뮬레이터가 필요해요

## 💡 유용한 팁

### Tip 1: Inspector Panel 활용

브라우저 탭 하단에 4가지 탭이 있어요:

1. **Elements**: HTML 구조 보기
2. **Console**: 로그, 에러, 경고 보기 (여기 중요!)
3. **Network**: API 요청 확인
4. **Element**: 선택한 요소 상세 정보 + [Add to Chat] 버튼

**추천**: Console 탭을 항상 켜두세요. 에러가 발생하면 즉시 확인할 수 있어요!

---

### Tip 2: 🐛 버튼 활용

**언제 사용하나요?**
- 에러가 여러 개 발생했을 때
- 어떤 에러가 먼저 발생했는지 확인하고 싶을 때
- AI에게 "전체 상황"을 보여주고 싶을 때

**사용 방법**:
1. Inspector Panel - Console 탭에서 로그 확인
2. 채팅 입력창의 🐛 버튼 클릭
3. "이 에러들 분석해줘" 또는 "우선순위대로 고쳐줘" 입력

---

### Tip 3: Element + Console 함께 사용

**강력한 조합**:
```
1. Inspect Mode로 버튼 선택 → [Add to Chat]
2. 🐛 버튼으로 Console 로그 추가
3. "이 버튼 클릭하면 왜 에러 나는지 고쳐줘" 입력

→ AI가 버튼 정보 + 에러 로그를 함께 보고 정확한 수정!
```

## 🚀 시작하는 방법

### 1단계: 프로젝트 열기
```
Octave에서 React/Next.js 프로젝트 열기
```

### 2단계: Dev Server 실행 (터미널)
```bash
npm run dev
```

### 3단계: 브라우저 탭 생성
```
- 중앙 영역 Tab Bar의 [+] 버튼 클릭
- "Browser" 선택
- URL 입력: http://localhost:3000
- Enter!
```

### 4단계: 개발 시작!
```
- 🔍 버튼으로 요소 선택
- 🐛 버튼으로 에러 디버깅
- AI에게 물어보기
- 코드 자동 수정!
```

## ❓ 자주 묻는 질문

### Q1: 브라우저 탭이 안 열려요!
**A**: Dev Server가 실행 중인지 확인하세요!
```bash
# 터미널에서 확인
npm run dev

# "Local: http://localhost:3000" 같은 메시지가 보여야 합니다
```

### Q2: Inspect Mode가 안 켜져요!
**A**: Chromium 기반 브라우저이므로 웹 페이지가 완전히 로드된 후에 사용하세요. 로딩 중이면 기다렸다가 다시 시도!

### Q3: Console 로그가 안 보여요!
**A**: Inspector Panel의 **Console 탭**을 클릭하세요. 기본적으로 숨겨져 있을 수 있어요.

### Q4: AI가 잘못된 코드를 만들어요!
**A**:
- Element를 정확히 선택했는지 확인
- Console 로그도 함께 전달하면 더 정확해요
- "이 부분만 고쳐줘"처럼 구체적으로 요청하세요

### Q5: 브라우저 탭 여러 개 열 수 있나요?
**A**: 네!
- localhost:3000 (메인 앱)
- localhost:6006 (Storybook)
- localhost:8080 (문서 사이트)
이렇게 여러 개 열고 전환 가능해요!
